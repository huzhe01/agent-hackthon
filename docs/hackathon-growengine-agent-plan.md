# GrowEngine 智投星 Agent 黑客松规划稿

> 面向 ToB 场景 AI Agent 赛道的参赛方案。基于现有 `adproto / GrowEngine` 项目改造，目标是在 2026 年 6 月 28 日 14:30 前形成可演示 Demo、路演 PPT、录屏视频和说明文档。

## 1. 一句话定位

GrowEngine 智投星 Agent 是面向电商商家、品牌增长团队和广告代运营公司的企业级投放运营 Agent，能够自动读取投放数据、诊断异常、生成调优方案，并在人工确认后完成广告计划创建、预算/出价调整和复盘报告生成。

## 2. 选择这个 ToB 痛点的原因

广告投放是一个典型的企业经营场景：预算真实、数据密集、决策频繁、ROI 压力明确。许多中小企业和服务商并不缺投放平台，而是缺一个能够持续解释数据、提出行动、降低试错成本的“投放运营副驾驶”。

现有项目已经具备广告计划管理、实时指标、智能诊断、竞价模拟、推荐系统和 AI 助手雏形，因此最适合从“广告平台 Demo”升级为“可执行的企业经营 Agent”。这样能同时体现产品完成度、场景创新性和技术深度。

## 3. 企业痛点定义

### 3.1 目标用户

- 中小电商商家：预算有限，缺专职投手，希望快速知道钱花得值不值。
- 品牌增长团队：投放计划多，渠道指标复杂，需要及时发现异常和增长机会。
- 广告代运营公司：同时服务多个客户，需要标准化诊断、执行和周报产出。

### 3.2 核心痛点

1. 数据看得见，但行动不明确  
   投放后台有大量 CTR、CVR、CPA、ROI、消耗、展示等指标，但运营人员仍要人工判断“该加预算、降出价、换素材还是暂停计划”。

2. 异常发现慢，错过预算窗口  
   计划学习失败、ROI 跌破盈亏线、预算消耗过快、优质计划起量不足等问题，如果不能及时处理，会直接造成预算浪费。

3. 新建计划门槛高，经验依赖强  
   新品推广、节日促销、库存清仓等场景需要选择目标、预算、出价、人群和创意，初级运营很难一次配对。

4. 调参缺少推演，试错成本高  
   企业不敢直接改预算和出价，因为不知道调整后可能带来的消耗、转化、ROI 和预算节奏变化。

5. 复盘报告机械重复  
   每天/每周都要把计划表现、问题原因、调优动作和下步策略整理成报告，耗时但高度模板化。

## 4. 产品方案

### 4.1 产品名称

推荐使用：GrowEngine 智投星 Agent

备选名称：
- 投放经营 Agent
- GrowthOps Agent
- 企业增长投放副驾驶

### 4.2 核心价值主张

从“看数据”升级为“Agent 帮你闭环经营动作”：  
看数 -> 诊断 -> 推演 -> 生成执行方案 -> 人工确认 -> 执行 -> 复盘。

### 4.3 Demo 主流程

建议录屏和路演围绕一个完整故事：

1. 运营负责人进入 GrowEngine 投放驾驶舱，看到 ROI、消耗、GMV 和计划状态。
2. 打开右侧“智投星 Agent”，询问：“今天哪些计划需要我处理？”
3. Agent 调用计划数据和诊断工具，发现：
   - 一个计划学习失败，建议调整定向或提高出价。
   - 一个计划 ROI 低于盈亏线，建议暂停或降预算。
   - 一个高 ROI 计划预算消耗不足，建议加预算抢量。
4. 用户追问：“帮我模拟把高 ROI 计划预算提高 20%、出价提高 15% 的效果。”
5. Agent 调用竞价模拟，展示预估消耗、转化、CPA、ROI 变化，并给出风险提示。
6. 用户说：“确认执行推荐方案。”
7. Agent 展示执行预览卡片，列出将要调整的计划、字段、原值、新值、预估影响，用户点击确认。
8. 系统更新计划，并在行动日志中记录“谁在何时基于什么建议执行了什么动作”。
9. 用户说：“生成一份今天的投放复盘。”
10. Agent 输出结构化复盘：总体表现、异常计划、已执行动作、后续建议。

## 5. MVP 功能范围

### 5.1 必做功能

- Agent 对话入口：复用现有右侧“智投星”悬浮面板。
- 工具调用可视化：展示 Agent 正在读取计划、诊断、模拟或生成执行预览。
- 投放健康诊断：读取广告计划数据，识别低 ROI、学习失败、高潜力起量不足。
- 计划生成预览：自然语言创建广告计划，保留“确认后创建”机制。
- 调参执行预览：预算、出价、状态调整必须先展示 diff，再由用户确认。
- 竞价模拟解释：把现有 `/api/bidding/simulate` 的结果转成业务语言。
- 复盘报告生成：生成可复制的日报/周报文本。

### 5.2 加分功能

- 行动日志：记录 Agent 建议、人工确认和实际执行结果。
- 企业安全护栏：高风险动作需要二次确认，例如暂停计划、预算上调超过 30%。
- 多角色视角：老板看 ROI 和预算，投手看 CPA/CTR/CVR，代运营看客户周报。
- 推荐系统联动：从推荐模块读取高 CTR 广告素材，建议补充到低表现计划中。

### 5.3 暂不做

- 接入真实广告平台 API。
- 复杂权限系统。
- 真实多租户账套。
- 训练新的推荐/竞价模型。
- 多渠道全量归因。

## 6. 技术改造建议

### 6.1 现有基础

项目已有能力：

- `backend/api.py`：FastAPI 后端，含广告计划 CRUD、实时指标、诊断、竞价模拟、Agent SSE 接口。
- `frontend/src/App.jsx`：React 投放后台，含驾驶舱、计划表、智能诊断、右侧 Agent 面板。
- `ad_rec_backend` / `ad_rec_frontend`：广告推荐系统，可作为素材/人群推荐加分点。
- `docker-compose.yml`：已有前后端部署基础。

### 6.2 推荐新增/强化的 Agent 工具

后端可扩展以下工具：

- `get_campaigns`：读取广告计划。
- `get_diagnosis`：读取智能诊断。
- `simulate_campaign_adjustment`：模拟预算/出价调整影响。
- `create_campaign_preview`：生成新计划预览。
- `update_campaign_preview`：生成调整计划的 diff 预览。
- `apply_campaign_changes`：用户确认后执行调整。
- `generate_report`：生成投放复盘报告。
- `get_recommended_creatives`：从推荐系统获取高潜力素材或广告。

### 6.3 推荐前端改造

- 在 Agent 面板内增加“工具调用时间线”：读取数据、诊断、模拟、生成预览、等待确认、执行完成。
- 计划预览卡片升级为“执行确认卡片”，展示原值、新值、预估影响、风险提示。
- 在广告计划表中增加“Agent 建议”标签，例如“建议加预算”“建议暂停”“建议换素材”。
- 新增“行动日志”抽屉或面板，体现企业级可审计性。

## 7. 产品创新点

1. 从通用聊天助手变成企业可执行 Agent  
   不只回答问题，而是能调用工具、生成执行方案，并在确认后修改业务对象。

2. 把广告投放黑盒变成可解释流程  
   Agent 不只说“建议加预算”，还解释原因、展示指标、调用模拟器推演影响。

3. 适合企业真实流程的人机协同  
   所有写操作必须经过人工确认，高风险动作有护栏，动作有日志可追溯。

4. 横跨数据分析、策略生成、执行和复盘  
   覆盖企业增长运营的完整闭环，比单点报表或单点聊天更完整。

5. 基于现有广告推荐和竞价模拟能力扩展  
   技术上不是单纯套壳 LLM，而是 LLM + 业务工具 + 投放数据 + 仿真能力的 Agent。

## 11. 推荐路演话术

开场：

> 企业投放团队每天都在看报表，但真正困难的不是看见数据，而是把数据变成及时、可信、可追溯的经营动作。GrowEngine 智投星 Agent 希望解决的就是这个问题。

产品说明：

> 它不是一个只会聊天的 AI 助手，而是一个能读取投放数据、诊断异常、模拟调参效果，并在人工确认后执行计划调整的企业级 Agent。

创新总结：

> 我们把广告投放从“人盯报表、人凭经验调参”，升级为“Agent 发现问题、解释原因、推演影响、人确认后执行”，同时保留企业场景必须的安全护栏和行动日志。

结尾：

> 今天的 Demo 基于模拟广告数据，但架构上已经可以接入真实投放平台 API。它可以成为中小商家和代运营团队的 AI 投放运营副驾驶。

- Demo 闭环完整，能展示从分析到执行。
- 商业化表达自然，适合争取最佳商业/价值奖。
